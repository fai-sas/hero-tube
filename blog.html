Discuss the scope of var, let, and const

Scope of Var

  • Only accessible within the function where it is defined.
  • If defined outside of any function, it is globally scoped and can be accessed from any part of the code.

Scope of Let

  • Only accessible within the block where it is defined.
  • If defined inside a function, it is only accessible within that function.
  • If defined outside of any function or block, it is not accessible from any part of the code.

Scope of Const

  • Only accessible within the block where it is defined.
  • If defined inside a function, it is only accessible within that function.
  • If defined outside of any function or block, it is globally scoped and can be accessed from any part of the code.
Tell us the use cases of null and undefined

Use Cases of Null

  • Null is used to identify the existence of any element or object. For example, if a variable or object property has not been declared or initialized, it will be null.
  • Null can be used to prevent errors when working with variables that may not have a value. For example, to check if any property of an object is available or not, property can be checked if it null or not.
  • Null can be used to return a default value when no value is available. For example, a function that doesn't take any arguments may return null by default.

Use Cases of undefined

  • When a variable is declared but not initialized, it will be undefined.
  • When a function does not have any meaningful return value it can return undefined.
  • Undefined will be returned if no parameter is passed in function.
  • If any property does not exist in an object will return undefined.
  • Accessing array elements outside of the index range will return undefined..
  • Deleting an element inside of an array will return undefined
What do you mean by REST API?

REST stands for Representational State Transfer. Rest API is used to Create, Read, Update and Delete elements in a website. Here are the most common methods used in REST APIs:

  • Get: Receive information about API resource
  • Post: Create an API resource
  • Put/Patch: Update an API resource
  • Delete: Delete an API resource
